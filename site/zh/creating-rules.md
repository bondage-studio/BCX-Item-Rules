本页介绍如何注册一个制作道具名称，并使用 BCXIR 的虚拟编辑流程为其附加 BCX 规则。

## 注册表

你创建的每条道具规则都保存在**本地注册表**中，按你的成员编号作为键：

```text
localStorage["BCXIR_registry_<MemberNumber>"]
```

每个条目包含：

| 字段 | 含义 |
| --- | --- |
| **名称** | 要匹配的精确制作道具名称。 |
| **启用** | 该条目是否参与匹配 / 分享。 |
| **仅自己** | 开启后，规则只对你生效，绝不分享给其他玩家。 |
| **规则数量** | payload 中包含的 BCX 规则条数。 |
| **更新时间** | 最近编辑的时间戳。 |

## 创建条目

1. 打开 **`BCXIR Settings` → 道具规则**。
2. 选择**创建 / 添加**，输入与游戏中完全一致的制作道具名称。
3. 系统会创建一个启用的新条目，带有一个合法但为空的 payload，随时可编辑。

在道具规则列表中，你还可以对条目执行**重命名**、**复制**、**启用 / 禁用**、**删除**、**匹配测试**和**同步**。

## 使用虚拟编辑角色编辑规则

选择 **Edit BCX Rules（编辑 BCX 规则）** 会启动一个编辑会话：

- BCXIR 激活一个**虚拟 BCX 通信层**，并创建一个临时的本地角色 **`BCXIR Authoring`**。
- 它打开该角色的**信息表 → Rules**，让你使用真正的 BCX 规则编辑器。
- 你在其中的所有操作都由内存中的虚拟规则存储**在本地**响应；不会向服务器发送任何内容，也绝不触碰你自己的 BCX。

在编辑器内，你可以使用编辑存储所支持的 BCX 规则功能，包括创建、删除和配置规则条件，以及 Rules 的全局配置。

当你选择 **Finish / Save（完成 / 保存）** 时：

- 虚拟规则会被导出为紧凑的 BCXIR payload。
- payload 会以**你正在编辑的道具名称**保存到注册表条目。
- 临时角色和虚拟通信层会被清理。

> **取消编辑。** 如果出现问题，你可以在 **调试 / 诊断 → 取消编辑（Cancel authoring）** 中取消。临时角色和通信层会被移除且不保存。

## 仅自己条目

如果规则只供你自己使用、绝不分享，请将条目的**仅自己（Self-only）**开关打开。仅自己条目：

- 当你穿戴匹配道具时仍会应用到你身上。
- **绝不**返回给请求该 payload 的其他玩家。

仅自己与分享如何交互，见[分享与权限](/zh/bcxir/sharing)。

## 运行时如何匹配

当某件制作道具被穿戴时，BCXIR 会根据**谁制作了它**来决定行为：

- **由你制作** → BCXIR 读取你的本地**注册表**并按配置应用规则。
- **由他人制作** → BCXIR 检查你的本地**缓存**，必要时向制作者请求 payload（见[分享](/zh/bcxir/sharing)）。

完整的运行时 / 恢复行为及存储细节，见[工作原理](/zh/bcxir/how-it-works)。
